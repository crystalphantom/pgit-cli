# Product

## Register

brand

## Users
AI-assisted software developers, command line power-users, and collaborative team members who need to keep local configuration files, notes, and credentials visible to their local IDE and AI agents while keeping them safely out of shared Git history.

## Product Purpose
A developer-focused landing page for PGit CLI, showcasing the core benefits of agent-visible private config tracking, hook-based git safety, and command workflows. It serves to convince developers to adopt and install the CLI tool.

## Brand Personality
Clean, technical, minimal, and premium. It emphasizes strong typographic hierarchy, spacious layouts, and subtle micro-animations. It avoids generic SaaS aesthetics in favor of a precise, high-end developer tool look.

## Anti-references
- SaaS cliché elements like identical card grids, large hero metrics, and tiny uppercase eyebrows above every section.
- Ghost cards combining 1px borders with soft, wide drop shadows.
- Saturated cream/beige backgrounds.
- Loud decorative text gradients or sketchy SVG doodle illustrations.

## Design Principles
- Developer-First Precision: Clear, highly readable terminal syntax, clean code examples, and structured layouts.
- Minimalist Clarity: Use whitespace, typographic scale, and solid background surfaces for structure instead of boxing everything in borders.
- Functional Motion: Smooth, subtle transitions that respond to user actions and respect accessibility settings without causing distraction.

## Accessibility & Inclusion
- Contrast ratios meeting WCAG AA standards (especially for code snippets and secondary text elements on dark backgrounds).
- Full keyboard navigation support and clear focus indicators.
- Respect for prefers-reduced-motion media queries.
